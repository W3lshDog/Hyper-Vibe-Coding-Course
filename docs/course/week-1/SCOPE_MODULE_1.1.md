# 📄 Scoping Document: Module 1.1 - The Vibe Coding Mindset

## 🎯 Objectives
- **Core Goal**: Shift student mindset from "coding = syntax" to "coding = taste + direction".
- **Outcome**: Student feels empowered, not intimidated. Understands their role as "Creative Director" of AI.
- **Engagement**: <15 min total time. High energy.

## 📦 Features & Deliverables

### 1. Video Lesson: "Your First Vibe" (8 min)
- **Format**: Loom/Screen recording with face cam.
- **Script**: [See CURRICULUM.md lines 50-108]
- **Visuals**:
  - Split screen: VS Code (scary syntax) vs. Claude Chat (natural language).
  - "The Old Way" vs. "The Vibe Way" comparison.
  - Montage of cool apps built with 0 code.

### 2. Reading: "The Vibe Coding Manifesto" (7 min)
- **Format**: Markdown file / Notion page / PDF.
- **Content**: [See CURRICULUM.md lines 110-154]
- **Key Points**:
  - Code is implementation detail.
  - Prompting > Programming.
  - Taste is the bottleneck.

### 3. Community Activation (Discord)
- **Prompt**: "What's ONE thing you want to build but thought you never could?"
- **Channel**: `#introductions`
- **Goal**: Public commitment + community bonding.

## 🏗️ Technical Architecture
- **Platform**: Hosted on Gumroad (Video embed + PDF download) or Course Platform (Teachable/Kajabi).
- **Assets**:
  - `VIDEO_1.1.mp4` (1080p, ~100MB)
  - `MANIFESTO.pdf` (Designed PDF)
  - `TRANSCRIPT_1.1.md` (Accessibility)

## 👤 User Stories
- **As a beginner**, I want to know why this course is different so that I don't feel overwhelmed by technical jargon immediately.
- **As a neurodivergent learner**, I want short, punchy content so that I stay engaged.
- **As a skeptic**, I want to see proof that "no code" actually works so that I trust the instructor.

## ✅ Acceptance Criteria
- [ ] Video audio is clear, no background noise.
- [ ] Video visuals clearly show the "Old vs New" comparison.
- [ ] Manifesto is readable (good typography, bold key points).
- [ ] Discord link in lesson description works.
- [ ] "Mark as Complete" button functions.

## 📅 Delivery Roadmap
- **Day 1**: Script refinement & slide deck creation.
- **Day 2**: Filming & Editing.
- **Day 3**: Asset creation (Manifesto PDF).
- **Day 4**: Upload & QA check.

## ⚠️ Risk Register
- **Risk**: Video is too long/boring.
  - *Mitigation*: Strict 8 min limit. Edit out pauses. Keep energy high ("BROski" tone).
- **Risk**: Students skip the mindset part to get to building.
  - *Mitigation*: Make the video required to unlock Module 1.2.
