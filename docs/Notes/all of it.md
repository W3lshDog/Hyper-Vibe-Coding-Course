🧩 Module 1 – Vibe Mindset + Setup (Why)
Why: You stop thinking “learn syntax” and start thinking “direct a smart teammate”.

Concepts: what is vibe coding, AI as pair dev, “describe → refine → ship” loop

Setup: accounts for Lovable + Replit, GitHub, basic project folders

Quick win: run your first AI-generated app in under 60 minutes
​

Example prompts:

“You are a friendly senior dev helping a beginner build tiny web apps with Lovable. Help me set up my first project step by step and explain each choice in simple terms.”

“Act as my ADHD-friendly coding coach. I get overwhelmed. Help me create a 30-minute micro-session to build a simple ‘daily mood logger’ app in Lovable.”

🛠️ Module 2 – Tiny Apps, Big Loop (How)
Why: The loop is your real teacher: describe → AI builds → you test → AI fixes.

You’ll practice the loop:

Write idea in plain language

Ask AI for structure

Generate first draft

Run → break → fix → improve UI → share
​

Hands-on builds (you pick 1–2):

Water tracker app

Daily mood or energy logger

“What should I do next?” decision helper

Example prompts:

“You are a senior frontend dev. Build a minimal water tracker web app with add/remove buttons, today’s total, and simple styling. Use a simple stack that Lovable can deploy easily.”

“I’m testing the app now. I found a bug: [paste error]. Explain what’s wrong and why, but don’t fix it yet. Help me understand it like I’m new to coding.”

🎯 Module 3 – Prompts = Your New Programming Language
Why: Better prompts → better code → fewer rewrites. Prompts become your “API design”.

You’ll learn to include:

Role + context + task

User/audience

Features + constraints

Aesthetic / vibe (e.g. “cozy Notion-style UI”)
​

Mini practice:

Turn vague prompts into sharp ones

Ask for alternatives (“give me 3 different UX takes”)

Add constraints (performance, accessibility, mobile-first)

Example prompts:

“You are a product-minded full-stack dev. For a focus timer app for ADHD users, propose 3 UI concepts, each with pros and cons, and pick one that’s simplest to ship this week.”

“Rewrite this vague prompt into a powerful one for Cursor or Lovable. Goal: ship a small, fast, mobile-friendly task list app with persistence and no login.”

🐛 Module 4 – Debugging WITH AI (Not Just Using It)
Why: This is where you stop being a button-clicker and start being a real builder.

You’ll practice:

Explaining errors in your own words

Asking “why” before “fix it”

Doing rubber-duck sessions where you narrate code to the AI

Patterns you’ll use:

“Explain what this error means and what part of the code it points to.”

“Walk me through this file line by line and point out anything risky or unclear.”
​

Example prompts:

“Here is my error and the code. Explain what’s wrong and why in simple terms, but don’t fix it yet. Then ask me 3 questions to see if I understand.”
​

“Act as my rubber duck. I will explain this component line by line. Stop me only if something is wrong or inefficient and ask clarifying questions.”
​
